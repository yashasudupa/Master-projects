Detailed Description

OTAWA provides many code processors supporting IPET approach:

  • VarAssignment - assign ILP variable to each basic block and edge of the graph.
  • BasicConstraintsBuilder - build constraints generated from the program CFG,
  • BasicObjectFunctionBuilder - build the object function to maximize to get the WCET,
  • ConstraintLoader - add constraints from an external file,
  • FlowFactLoader - load and add constraints for the flow facts (currently constante loop upper bounds only),
  • WCETComputation - perform the WCET final computation.

Some processors are dedicated to pipeline-level analyses:

  • TrivialBBTime - compute the execution time of a basic block by applying a constant execution to each instruction,
  • Delta - change the constraints and the object function to implement the Delta method for inter-basic-block pipe-line effect support,
  • ExeGraphBBTime - compute the execution time of basic block using an execution graph.

Some processors are dedicated to cache analyses:

  • CATBuilder - compute the cache category of each instruction (always-miss, always-hit, first-miss, first-it),
  • CATConstraintBuilder - add constraints to use instruction access categories,
  • CCGBuilder - build the Cache Conflict Graph for the instruction cache,
  • CCGConstraintBuilder - build the constraints using the CCG to take in account instruction cache accesses,
  • CCGObjectFunction - the CCG instruction cache approach requires to use a special object function in order to get the WCET,
  • TrivialDataCacheManager - consider each memory access as a miss.

These processors interacts using the following properties:

  • TIME - execution time of a program part (usually a basic block),
  • VAR - ILP variable associated with a program component (usually a basic block or an edge),
  • SYSTEM - ILP system associated with a root CFG,
  • WCET - WCET of a root CFG,
  • EXPLICIT - used in configuration of code processor, cause the generation of explicit variable names instead of numeric ones (default to false),
  • RECURSIVE - cause the processors to work recursively (default to false).
